# Plan 11-01: Backend — Price Alerts

## Overview

Add the `price_alerts` SQLite table, REST CRUD endpoints, a background check loop that fires alerts and pushes them onto an `asyncio.Queue`, an `event: alert_fired` SSE event on the existing price stream, and fired-alert context injection into `_portfolio_context()`.

## must_haves

- `POST /api/alerts` creates a row; `GET /api/alerts` returns it; `DELETE /api/alerts/{id}` removes it
- Background loop sets `active=0, triggered_at=<iso>` and puts event on queue within 500ms of price crossing threshold
- SSE stream emits `event: alert_fired\ndata: {...}\n\n` within one loop cycle of the queue item arriving
- `_portfolio_context()` includes a "Recent price alerts fired (last 30 min)" block when fired alerts exist
- All 7 unit tests in `test_alerts.py` pass

## Task 11-01-05: Add `_alert_check_loop` background task in `main.py`

<read_first>

- backend/app/db.py                 (get_db, price_alerts table schema)
- backend/app/market/stream.py      (create_stream_router now accepts alert_queue)
</read_first>

<action>
In `backend/app/main.py`:

1. Add `_alert_check_loop` async function above `lifespan`:

    async def _alert_check_loop(price_cache: PriceCache, alert_queue: asyncio.Queue) -> None:
        while True:
            await asyncio.sleep(0.5)
            try:
                with get_db() as conn:
                    rows = conn.execute(
                        "SELECT id, ticker, target_price, direction FROM price_alerts "
                        "WHERE user_id='default' AND active=1"
                    ).fetchall()
                for row in rows:
                    current = price_cache.get_price(row["ticker"])
                    if current is None:
                        continue
                    fired = (
                        row["direction"] == "above" and current >= row["target_price"]
                    ) or (
                        row["direction"] == "below" and current <= row["target_price"]
                    )
                    if not fired:
                        continue
                    now = datetime.now(timezone.utc).isoformat()
                    with get_db() as conn2:
                        conn2.execute(
                            "UPDATE price_alerts SET active=0, triggered_at=? WHERE id=?",
                            (now, row["id"]),
                        )
                    await alert_queue.put({
                        "id": row["id"],
                        "ticker": row["ticker"],
                        "target_price": row["target_price"],
                        "direction": row["direction"],
                        "current_price": current,
                        "triggered_at": now,
                    })
            except Exception:
                logger.exception("Alert check error")

2. In `lifespan()`, add before the `yield`:
    - `alert_queue: asyncio.Queue = asyncio.Queue()`
    - `app.state.alert_queue = alert_queue`
    - `alert_task = asyncio.create_task(_alert_check_loop(_price_cache, alert_queue), name="alert-check")`
    - Change `create_stream_router(_price_cache)` call to `create_stream_router(_price_cache, alert_queue)`

3. In `lifespan()` cleanup (after `yield`), add:
    - `alert_task.cancel()`
    - `try: await alert_task\nexcept asyncio.CancelledError: pass`

4. Add `from datetime import datetime, timezone` at top of `main.py` if not already present.
</action>
