# Plan 11-02: Frontend — Price Alerts UI

## Overview

Wire the frontend to the backend alerts API: types, API functions, `useAlerts` hook, `useMarketData` SSE extension, `ToastContainer` component, `AlertPopover` popover, bell icon in `WatchlistPanel`, and `page.tsx` wiring. Zero new npm dependencies.

## must_haves

- Bell icon appears on watchlist row hover; clicking it opens `AlertPopover`
- `AlertPopover` submits `POST /api/alerts` and the new alert appears in `useAlerts` state
- `event: alert_fired` SSE event triggers `onAlertFired` callback → toast appears bottom-right within 1 second
- Toast auto-dismisses after 4 seconds and dismisses on click
- Bell icon fills amber when ticker has at least one active alert
- Alerts persist across page refresh (loaded from `GET /api/alerts` on mount)

## Task 11-02-03: Create `frontend/hooks/useAlerts.ts`

<read_first>

- frontend/lib/api.ts               (fetchAlerts, createAlert, deleteAlert)
- frontend/lib/types.ts             (Alert, FiredAlert)
</read_first>

<action>
Create `frontend/hooks/useAlerts.ts`:

Exports function `useAlerts()` with `"use client"` directive.

State: `alerts: Alert[]` (default `[]`), `toasts: FiredAlert[]` (default `[]`).

`refresh` — `useCallback` that calls `fetchAlerts()` and sets `alerts`, silently ignores errors.

`useEffect` on mount calls `refresh()`.

`handleAlertFired(fired: FiredAlert)` — removes the alert from `alerts` (filter by `id`), adds to `toasts`, sets a 4000ms `setTimeout` to remove it from `toasts`.

`dismissToast` — `useCallback` that removes a toast by `id` from `toasts` immediately (for click-to-dismiss).

`createAlert(ticker, targetPrice, direction)` — `useCallback` that calls `apiCreateAlert`, appends the returned alert to `alerts`.

`removeAlert(alertId)` — `useCallback` that calls `apiDeleteAlert`, filters alert out of `alerts`.

Return: `{ alerts, toasts, handleAlertFired, dismissToast, createAlert, removeAlert }`.

Note: no setInterval polling — alerts are loaded once on mount and updated imperatively. This avoids stale closure issues and unnecessary re-renders.
</action>

<acceptance_criteria>
- `frontend/hooks/useAlerts.ts` exists and exports `useAlerts` function
- `useAlerts` returns `{ alerts, toasts, handleAlertFired, dismissToast, createAlert, removeAlert }`
- `handleAlertFired` filters the fired alert out of `alerts` and adds it to `toasts`
- `toasts` entry auto-removes after 4000ms via `setTimeout`

Modify `frontend/hooks/useMarketData.ts`:

2. Change the hook signature to: `export function useMarketData(options?: { onAlertFired?: (alert: FiredAlert) => void })`.

3. Add a `useRef` to hold the latest callback without it becoming a dependency:

    const onAlertFiredRef = useRef(options?.onAlertFired);
    useEffect(() => { onAlertFiredRef.current = options?.onAlertFired; }, [options?.onAlertFired]);

4. Inside the `connect` callback, after `es.onmessage = ...`, add:

    es.addEventListener("alert_fired", (event: MessageEvent) => {
      try {
        const fired: FiredAlert = JSON.parse(event.data);
        onAlertFiredRef.current?.(fired);
      } catch { /* ignore parse errors */ }
    });

5. The `connect` `useCallback` dependency array does NOT include `options` or `onAlertFired` directly (uses the ref to avoid re-creating EventSource on every render).
</action>
