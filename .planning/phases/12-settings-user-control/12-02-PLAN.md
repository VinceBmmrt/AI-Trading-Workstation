# Plan 12-02: Frontend — Settings Panel, Theme Toggle & Layout Persistence

## Overview

Add `AppSettings` type, API functions, light-theme CSS overrides, `useTheme` hook, `SettingsPanel` component, localStorage persistence for four state variables, and a gear icon in the Header. Depends on Wave 1 backend.

- Portfolio reset in SettingsPanel calls `POST /api/portfolio/reset` with two-step inline confirm
- `selectedTicker`, `chatOpen`, `portfolioTab`, `mobilePanel` all restore from localStorage on page reload
- `cd frontend && npx tsc --noEmit` passes; `npm run build` succeeds

## Task 12-02-04: Create `frontend/hooks/useTheme.ts`

<read_first>
- frontend/hooks/useAlerts.ts    (hook structure: "use client", useState with lazy init, useEffect, useCallback)
- frontend/app/globals.css       (confirms `html.light` is the class to toggle)
</read_first>

<action>
Create `frontend/hooks/useTheme.ts` (new file):

1. `"use client";` directive at top.

2. Import `useCallback`, `useEffect`, `useState` from `"react"`.

3. Export named function `useTheme()`:

   a. State with SSR-safe lazy initializer:
      `const [theme, setTheme] = useState<"dark" | "light">(() => { if (typeof window === "undefined") return "dark"; return (localStorage.getItem("fa_theme") as "dark" | "light") ?? "dark"; });`

   b. `useEffect` with `[theme]` dependency:
      - `document.documentElement.classList.toggle("light", theme === "light");`
      - `localStorage.setItem("fa_theme", theme);`

   c. `const toggleTheme = useCallback(() => setTheme((t) => (t === "dark" ? "light" : "dark")), []);`

   d. Return `{ theme, setTheme, toggleTheme }`.

Theme is client-side only (localStorage + DOM class). No API calls in this hook.
</action>

<acceptance_criteria>
- `useTheme()` returns `{ theme, setTheme, toggleTheme }`
- Calling `toggleTheme()` switches `theme` from `"dark"` to `"light"` and vice versa
- When `localStorage.getItem("fa_theme")` returns `"light"`, `theme` initializes to `"light"`
- When `theme === "light"`, `document.documentElement.classList.contains("light")` is `true`
- When `theme === "dark"`, `document.documentElement.classList.contains("light")` is `false`
- `cd frontend && npx tsc --noEmit` passes
</acceptance_criteria>

- Label: `<div className="text-[10px] text-text-dim uppercase tracking-widest mt-3 mb-1">Starting Capital</div>`

**Error display:** `{error && <p className="text-down text-[10px] mt-1">{error}</p>}`

**Section 3 — Reset Portfolio:**

- When `!confirmReset`: button `"Reset Portfolio"` with `bg-down/10 text-down border border-down/30` → `onClick={() => setConfirmReset(true)}`
- When `confirmReset`: warning text `<p className="text-[10px] text-text-dim mb-2">This will wipe all positions, trades and snapshots.</p>` + two buttons:
  - `"Confirm Reset"`: `bg-down/20 text-down border border-down/40`, `disabled={resetLoading}` → `onClick` calls `resetPortfolio()`, sets `resetLoading` true/false, on success calls `onResetComplete()` then `onClose()`, on error sets `error`
  - `"Cancel"`: `bg-bg text-text-dim border border-border` → `onClick={() => setConfirmReset(false)}`
