- backend/app/db.py               (get_db import path)

<action>
Create `backend/app/routes/settings.py`. Structure mirrors `alerts.py`.

Constants: `USER_ID = "default"`.

Router: `APIRouter(prefix="/api/settings", tags=["settings"])`.

Pydantic model `SettingsUpdateBody(BaseModel)`: field `starting_capital: float` with a validator that raises `ValueError("starting_capital must be > 0")` when value <= 0. Use `@field_validator` (Pydantic v2) or `@validator` (Pydantic v1) — check which Pydantic version is in use by reading `backend/pyproject.toml`.

`GET ""` handler `get_settings()`:
- Opens `with get_db() as conn:` block
- Executes `SELECT starting_capital FROM app_settings WHERE id=?` with `(USER_ID,)`
- If no row: returns `{"starting_capital": 10000.0}`
- Otherwise: returns `{"starting_capital": row["starting_capital"]}`

`PUT ""` handler `update_settings(body: SettingsUpdateBody)`:
- Opens `with get_db() as conn:` block
- Executes `UPDATE app_settings SET starting_capital=? WHERE id=?` with `(body.starting_capital, USER_ID)`
- Returns `{"starting_capital": body.starting_capital}`

The Pydantic validator handles `<= 0` — FastAPI returns HTTP 422 automatically. Do NOT add a manual `HTTPException(422)`.
</action>

- `PUT /api/settings` with `{"starting_capital": 0}` returns HTTP 422

## Task 12-01-03: Add `POST /api/portfolio/reset` to `portfolio.py`

<read_first>

- backend/app/db.py                  (get_db context manager behaviour)

<action>
In `backend/app/routes/portfolio.py`, add a new route handler `@router.post("/reset")` with function name `reset_portfolio()`. Place it after the `get_trades` function and before the `STARTING_CAPITAL` constant line.

The handler body:
1. Wraps everything in `try/except Exception as e:` → `raise HTTPException(status_code=500, detail=f"Reset failed: {e}")` on error
2. Opens a single `with get_db() as conn:` block containing in order:
   a. `settings_row = conn.execute("SELECT starting_capital FROM app_settings WHERE id='default'").fetchone()`
   b. `starting = settings_row["starting_capital"] if settings_row else 10000.0`
   c. `conn.execute("DELETE FROM trades WHERE user_id=?", (USER_ID,))`
   d. `conn.execute("DELETE FROM positions WHERE user_id=?", (USER_ID,))`
   e. `conn.execute("DELETE FROM portfolio_snapshots WHERE user_id=?", (USER_ID,))`
   f. `conn.execute("UPDATE users_profile SET cash_balance=? WHERE id=?", (starting, USER_ID))`
3. Returns `{"success": True, "cash_balance": starting}`

Do NOT import `Request` — this handler does not need `price_cache`.
Do NOT wipe `chat_messages` or `price_alerts`.
</action>

- After reset: `SELECT COUNT(*) FROM trades WHERE user_id='default'` returns 0
- After reset: `SELECT COUNT(*) FROM portfolio_snapshots WHERE user_id='default'` returns 0
- After reset: `SELECT cash_balance FROM users_profile WHERE id='default'` returns 10000.0
- `chat_messages` rows are NOT deleted
- `price_alerts` rows are NOT deleted
