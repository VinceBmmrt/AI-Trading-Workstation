<task id="09-01-T01" type="execute">
  <title>Create frontend/lib/indicators.ts with calcMA and calcRSI</title>
  <read_first>
    - frontend/lib/types.ts (no changes needed — reference only for context)
  </read_first>
  <action>
    Create frontend/lib/indicators.ts with two exported pure functions:

    calcMA(prices: number[], period: number): (number | null)[]
    - For each index i < period-1: return null
    - For i >= period-1: return average of prices[i-period+1 .. i] inclusive
    - Example: calcMA([1,2,3,4,5], 3) → [null, null, 2, 3, 4]

    calcRSI(prices: number[], period = 14): (number | null)[]
    - Return array of same length as prices
    - Indices 0..period-1: return null (not enough data)
    - Index period: compute initial avgGain/avgLoss over first `period` changes
    - Indices period+1..: apply Wilder's smoothing
      avgGain = (avgGain*(period-1) + max(diff,0)) / period
      avgLoss = (avgLoss*(period-1) + max(-diff,0)) / period
    - RSI = avgLoss === 0 ? 100 : 100 - 100/(1 + avgGain/avgLoss)
    - If prices.length < period+1: return all nulls

    No imports needed — pure TypeScript functions only.
  </action>
  <acceptance_criteria>
    - frontend/lib/indicators.ts exists and exports calcMA and calcRSI
    - calcMA([1,2,3,4,5], 3) returns [null, null, 2, 3, 4]
    - calcMA([10], 3) returns [null]
    - calcRSI with all positive diffs (all gains) returns 100 at index period
    - calcRSI with all negative diffs (all losses) returns 0 at index period
    - calcRSI([10,11], 14) returns [null, null] (not enough data)
    - cd frontend && npx tsc --noEmit passes with no new errors
  </acceptance_criteria>
</task>
