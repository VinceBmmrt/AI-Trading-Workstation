# Phase 10 — Wave 1: Backend AI Intelligence

## Goal

Make the backend AI proactive and context-aware: track per-ticker session baselines, run a
background monitor that posts AI commentary on >2% session moves, expose a polling endpoint and
a market-summary endpoint, and enrich the LLM portfolio context. All LLM calls follow the
cerebras skill pattern via async `acompletion`.

## Context

- `backend/app/routes/chat.py` already defines `MODEL`, `EXTRA_BODY`, `MOCK_RESPONSE`, `LLMResponse`,
  `_portfolio_context(price_cache)`, `_chat_history()`, `_save_message(role, content, actions)`, and
  the `POST /api/chat` handler. Reuse these; do NOT duplicate constants.
- `backend/app/main.py` has the `lifespan` context manager that calls `source.start(tickers)`,
  sets `app.state.price_cache` and `app.state.market_source`, and starts `_snapshot_loop` via
  `asyncio.create_task`. Add the new monitor task in the same place with the same cancel-on-shutdown pattern.
- `PriceUpdate.change_percent` is TICK-TO-TICK (last 500ms) — it MUST NOT be used for session threshold
  detection. Session % must be computed as `(current_price - baseline) / baseline * 100` using
  `app.state.session_baselines`.
- `app.market.seed_prices.SEED_PRICES` holds simulator start prices, but the chosen approach is to
  snapshot the live cache once after `source.start()` so it works identically for simulator and Massive modes.
- `_save_message` already serializes the `actions` dict to JSON; proactive messages set
  `actions={"proactive": True, "ticker": <TICKER>, "trades": [], "trade_errors": [], "watchlist_changes": []}`.
- Starting capital constant is `10000.0` (matches `users_profile.cash_balance` default and `portfolio.py`).
