# Phase 10 — Wave 2: Frontend AI Intelligence

## Goal

Surface the Wave 1 backend capabilities in the UI: poll for and render proactive market-alert
messages, add a market-summary banner under the header, and support `/analyze`, `/rebalance`,
`/risk` slash commands with an autocomplete hint. Depends on the Wave 1 endpoints
(`GET /api/chat/messages`, `GET /api/chat/market-summary`) and enriched context.

## Context

- `frontend/lib/types.ts` defines `ChatActions` and `ChatMessage`. Extend `ChatActions` (do not break
  existing `trades`/`trade_errors`/`watchlist_changes` consumers in ChatPanel).
- `frontend/lib/api.ts` uses `const BASE = process.env.NEXT_PUBLIC_API_BASE ?? ""` and a `fetch` pattern;
  reuse that style and `BASE` for the new calls. `sendChat(message)` already exists — keep its signature.
- `frontend/components/ChatPanel.tsx` is a client component. Local `messages` state items are
  `(ChatMessage & { ts?: string })`. The first item has `id: "welcome"`. `handleSend` reads
  `input.trim()` into `msg`, appends a user bubble, calls `sendChat(msg)`, appends the assistant reply,
  and calls `onTradeComplete()` when trades/watchlist changes occurred. Auto-scroll runs on
  `[messages, loading]`. There is an existing action-chips block keyed off `msg.actions`.
- `frontend/app/page.tsx` renders `<Header .../>` then a `<div className="flex flex-1 ...">` main grid.
  The banner goes between them. It is a client component (`"use client"`).
- `frontend/AGENTS.md` warns this Next.js differs from training data — consult
  `node_modules/next/dist/docs/` before writing component code if anything is uncertain.
- Backend `/api/chat/messages` returns ascending-by-`created_at` items `{id, role, content, actions, created_at}`
  and accepts `?after_ts=<ISO>`. `/api/chat/market-summary` returns `{summary, generated_at}`.
